Przekazanie dla agenta aplikacji mobilnej (estateos-app)
========================================================

Kontekst: backend (Next na estateos.pl) serwuje wizytówki pod /o/:id, .well-known z ENV,
middleware HTTPS i rewrite są po stronie serwera — aplikacja tylko musi udostępniać
DOBRY link i treść promującą platformę (udostępnienie idzie do SMS / Messenger / Mail / itd.).

1) Co zmienić w aplikacji
-------------------------
Plik typowo: src/screens/OfferDetail.tsx — funkcja handleShare (Share.share).

• message (wielolinijkowy), minimum:
  — krótki opis: tytuł + cena oferty
  — kanoniczny URL: https://estateos.pl/o/<offerId>
  — linia promocyjna platformy, np.:
    „Szczegóły i negocjacje w aplikacji EstateOS™ — https://estateos.pl”

• iOS: jeśli obsługiwane, dodaj pole url: `https://estateos.pl/o/${offer.id}` przy Share.share,
  żeby Messages / Mail ładnie skleiły podgląd linku.

• Opcjonalnie wydziel do src/utils/offerShareUrls.ts:
  — buildOfferPublicCardUrl(id) → https://estateos.pl/o/id
  — buildOfferShareMessage(offer) → string gotowy do Share

• Zachować spójność z deeplinkiem aplikacji: estateos://o/<id> (już na landing page i w backendzie).

2) Jak tester ma to sprawdzić w UI
-----------------------------------
1. Zaloguj się w aplikacji.
2. Wejdź w dowolną ofertę (lista Radar / Moje oferty / wynik wyszukiwania).
3. Tap ikona UDOSTĘPNIJ (Share) na ekranie szczegółów oferty.
4. Wybierz cel (Notatki, Wiadomość, Mail) — w treści MA być https://estateos.pl/o/<id>
   oraz linia promująca EstateOS / estateos.pl.
5. Z phone otwórz ten sam URL w Safari — powinna załadować się publiczna wizytówka z estateos.pl.

3) Czego agent mobilny NIE musi robić na backendzie
----------------------------------------------------
Team ID / SHA-256 / nginx — serwerowy agent lub właściciel serwera ustawia w .env i deploy.
Aplikacja tylko korzysta z publicznego HTTPS.
